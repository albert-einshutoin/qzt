## 運用ルール (共通)

```text
1. 1 issue = 1 PR。マージ条件は make check green。
2. リファクタ PR は挙動を変えない。挙動が変わる issue (#3, #8, #22) は
   CHANGELOG.md に記録する。
3. マージ済み機能だけをドキュメントに書く。未実装の宣伝をしない。
4. docs やチュートリアルに書くコマンド・出力例は、すべて実行してから書く。
5. 非可逆な対外アクション (crates.io publish、v0.1.0 タグ) はオーナーの
   明示承認後にのみ実行する。
6. issue 内の行番号は commit 024c6c2 時点。行番号ではなくシンボル名で
   検索すること。
```

## これらのトラック外の繰り越し (v0.2 候補)

どちらのロードマップでも閉じない既知の制限。着手前に format / extension
version の判断と新たなオーナー判断が必要です。

```text
- sidecar 構築メモリ: index 構築が posting map 全体をメモリに保持
  (42 MB コーパスで約 0.6〜1.3 GB)
- sidecar サイズ: QZI は非圧縮の MVP 構造 (現実的なログコーパスで原文の
  約 2.1 倍)
- normalized search: SearchIndexSource::NormalizedUtf8 (Unicode 正規化、
  case folding、width folding)
- phrase / positional 検索セマンティクス (現在の multi-token 検索は
  co-occurrence)
- メンテナンスコマンド: repack / merge / compact
- embedded qzt-search-block-v1
- #45 スコープ外の競合比較: Tantivy、Lucene、seekable zstd、
  split zstd frames
```
